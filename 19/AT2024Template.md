## 1. Введение

### 1.1 Цели
Целью данного документа является определение требований к Системе учета офисного имущества. Система предназначена для автоматизации процессов учета предметов в офисных помещениях, отслеживания их текущей стоимости с учетом износа и своевременного списания предметов, состояние которых достигло 0%.

### 1.2 Границы применения
Документ распространяется на программное обеспечение "Системы учета офисного имущества", включая модуль учета кабинетов и предметов, модуль расчета износа и текущей стоимости, модуль списания предметов и веб-интерфейс для взаимодействия пользователей с системой.

### 1.3 Термины и аббревиатуры

| Термин/Аббревиатура | Определение |
|---------------------|-------------|
| Предмет | Любой объект учета в офисе (стол, стул, монитор, оргтехника и т.д.) |
| Кабинет | Офисное помещение, за которым закреплены предметы |
| Износ | Уменьшение стоимости предмета со временем, выражается в процентах |
| Текущая стоимость | Стоимость предмета на данный момент с учетом износа |
| Состояние | Оставшийся процент полезности предмета (100% = новый, 0% = подлежит списанию) |
| Списание | Процедура удаления предмета из активного учета при достижении его состояния 0% |
| Инвентаризация | Процедура проверки фактического наличия имущества без списания |
| Комиссия по утилизации | Группа сотрудников, уполномоченных принимать решение о списании |
| Комплектация | Процесс объединения/разъединения предметов с сохранением истории |

### 1.4 Ссылки
| Обозначение | Расшифровка |
|-------------|-------------|
| [IEEE-830] | IEEE Std 830-1998 |

### 1.5 Краткий обзор
Документ структурирован согласно стандарту [IEEE-830] и включает в себя введение, общее описание системы, детальные функциональные и нефункциональные требования.

## 2. Общее описание

### 2.1 Описание изделия
| Аспект | Описание |
|--------|----------|
| Интерфейсы системы | Система предоставляет данные о предметах и кабинетах через внутренний API |
| Интерфейсы пользователя | Веб-интерфейс для просмотра списка кабинетов и предметов с их состоянием и текущей стоимостью |
| Интерфейсы аппаратных средств ЭВМ | Система должна работать на стандартной серверной инфраструктуре |
| Интерфейсы программного обеспечения | Реляционная база данных для хранения информации о кабинетах, предметах и истории изменений |
| Интерфейсы коммуникаций | Не предусмотрено внешних интеграций на начальном этапе |
| Ограничения памяти | Подлежат уточнению на этапе проектирования |
| Действия | Автоматический пересчет стоимости и состояния предметов, пометка предметов для списания |

### 2.2 Функции изделия
| № | Функция |
|---|---------|
| 1 | Учет и каталогизация всех предметов в офисных помещениях |
| 2 | Закрепление предметов за конкретными кабинетами |
| 3 | Расчет и применение износа к стоимости предметов по заданным правилам |
| 4 | Автоматическое обновление текущей стоимости и состояния предметов |
| 5 | Отображение актуального списка предметов в кабинете |
| 6 | Функция списания предметов, достигших состояния 0% |
| 7 | Формирование отчетов по общей стоимости активов и списанным предметам |
| 8 | Перенос предметов между кабинетами с сохранением истории перемещений |
| 9 | Проведение инвентаризации с фиксацией расхождений без обязательного списания |
| 10 | Голосование комиссии по утилизации с требованием единогласного решения |
| 11 | Разборка и сборка комплектов предметов с сохранением истории происхождения |

### 2.3 Характеристики пользователей
| Роль | Описание |
|------|----------|
| Сотрудники | Просматривают перечень имущества в своем кабинете |
| Бухгалтеры | Ведут основной учет, вводят первоначальную стоимость, правила износа, проводят списание |
| Администраторы системы | Управляют пользователями, настраивают систему |
| Члены комиссии | Участвуют в голосовании по утилизации предметов |

### 2.4 Ограничения
| № | Ограничение |
|---|-------------|
| 1 | Система является внутренней и не предполагает интеграции с внешними бухгалтерскими системами на первом этапе |
| 2 | Правила расчета износа предполагаются линейными |

### 2.5 Предположения и зависимости
| № | Предположение/Зависимость |
|---|---------------------------|
| 1 | Каждый предмет имеет первоначальную стоимость, дату ввода в эксплуатацию и коэффициент износа |
| 2 | Износ начисляется периодически (ежедневно, ежемесячно) |
| 3 | Когда состояние предмета достигает 0%, он автоматически помечается как подлежащий списанию |
| 4 | Решение о списании требует единогласного одобрения комиссии |

## 3. Детальные требования

### 3.1 Функциональные требования

#### Атрибуты функциональных требований

| Атрибут | Описание |
|---------|----------|
| ID | Уникальный идентификатор требования |
| Наименование | Краткое описание сути требования |
| Важность | M (обязательное), S (желательное), C (возможное) |
| Стабильность | S (стабильное), M (модифицируемое), D (динамичное) |
| Описание | Детальное описание поведения системы |

#### Полная схема идентификаторов

| Префикс | Расшифровка | Роль |
|---------|-------------|------|
| F.EX- | Functional - Executor | Исполнитель |
| F.MG- | Functional - Manager | Руководитель |
| F.AD- | Functional - Administrator | Администратор |
| F.SYS- | Functional - System | Система |
| F.COM- | Functional - Commission | Член комиссии |

#### Требования для роли "Администратор/Бухгалтер"

| ID | Наименование | Важность | Стабильность | Описание |
|---|---|---|---|---|
| F.AD-001 | Просмотр кабинетов и предметов | M | S | Система предоставляет возможность просмотра списка всех кабинетов. При выборе кабинета отображается таблица со списком всех предметов в нем, их состоянием (в %) и текущей стоимостью |
| F.AD-002 | Расчет износа предметов | M | S | Система автоматически пересчитывает текущую стоимость и состояние каждого предмета на основе заданного коэффициента износа и периода по формуле: Текущая стоимость = Первонач. стоимость × (1 - (Коэф. износа × Время)) |
| F.AD-003 | Списание предметов | M | S | При достижении состоянием предмета 0% система помечает его как "Готов к списанию". Администратор/бухгалтер имеет возможность подтвердить списание предмета |
| F.AD-004 | Подтверждение списания | M | S | Система предоставляет интерфейс для подтверждения списания предметов, отмеченных как "Готов к списанию" |
| F.AD-005 | Перенос предметов между кабинетами | M | S | Система предоставляет возможность перемещения предметов из одного кабинета в другой с обязательным указанием даты перемещения и ответственного лица |
| F.AD-006 | Проведение инвентаризации | M | S | Система позволяет проводить инвентаризацию - фиксировать фактическое наличие предметов без обязательного списания. Отмечаются расхождения между учетными и фактическими данными |
| F.AD-007 | Управление комиссией по утилизации | M | S | Система предоставляет интерфейс для формирования комиссии по утилизации и проведения голосования по списанию предметов |
| F.AD-008 | Разборка/сборка комплектов | M | S | Система позволяет разбирать комплектные предметы на составляющие (например, компьютер на монитор и системный блок) и собирать комплекты из отдельных предметов |

#### Требования для роли "Система"

| ID | Наименование | Важность | Стабильность | Описание |
|---|---|---|---|---|
| F.SYS-001 | Автоматический расчет износа | M | S | Система автоматически выполняет пересчет текущей стоимости и состояния каждого предмета по расписанию или при изменении данных |
| F.SYS-002 | Маркировка предметов для списания | M | S | Система автоматически помечает предметы как "Готов к списанию" при достижении состоянием 0% |
| F.SYS-003 | Ведение истории перемещений | M | S | Система автоматически сохраняет полную историю перемещений каждого предмета между кабинетами с указанием дат и ответственных лиц |
| F.SYS-004 | Контроль голосования комиссии | M | S | Система требует единогласного решения всех членов комиссии для подтверждения утилизации предмета |
| F.SYS-005 | Управление комплектацией | M | S | Система сохраняет историю происхождения предметов при разборке/сборке комплектов, позволяя отслеживать из каких компонентов был собран предмет |

#### Требования для роли "Член комиссии"

| ID | Наименование | Важность | Стабильность | Описание |
|---|---|---|---|---|
| F.COM-001 | Голосование за утилизацию | M | S | Система предоставляет интерфейс для голосования за утилизацию предметов. Каждый член комиссии может проголосовать "За" или "Против" |
| F.COM-002 | Просмотр истории предметов | M | S | Система предоставляет доступ к полной истории предмета (перемещения, ремонты, изменения стоимости) для принятия обоснованного решения |

# 1. Введение
Данный документ представляет собой спецификацию требований к программному обеспечению для системы автоматизированного сбора и анализа метрик футбольного матча. В документе описаны цели, функции, ограничения и интерфейсы системы, предназначенной для повышения качества тренировочного процесса и тактической подготовки футбольных команд.

## Цели
Цель системы - обеспечить автоматизированный сбор, анализ и визуализацию метрик футбольного матча для повышения качества тренировочного процесса и тактической подготовки команды. Система позволяет отслеживать индивидуальные и командные показатели, оценивать физическую нагрузку игроков, создавать тепловые карты перемещений и выявлять игровые ситуации, требующие анализа.

## Границы применения
Документ описывает программное обеспечение облачной аналитической платформы, которая:

- Агрегирует данные с носимых датчиков движения
- Анализирует данные для формирования индивидуальных и командных метрик
- Визуализирует данные в виде отчетов, графиков и тепловых карт

Система предназначена для использования профессиональными и любительскими футбольными клубами, спортивными школами, тренерским и медицинским штабом.

## Термины и аббревиатуры
- **Sh90** - количество ударов по воротам в среднем за 90 минут
- **Fls** - количество совершённых фолов игроком
- **Int** - количество перехватов мяча
- **Aerial Duels** - количество единоборств в воздухе
- **Владение мячом (%)** - доля времени, когда игрок контролировал мяч
- **Тепловая карта (heatmap)** - визуализация активности перемещений игрока на поле
- **Датчик трекинга** - сенсор, закрепляемый на теле игрока для фиксации положения

# 2. Общее описание

## Описание изделия
Система представляет собой комплекс аппаратных датчиков и программного обеспечения для сбора и анализа данных о действиях и состояниях футболистов. Датчики, закреплённые на игроках, фиксируют перемещения и игровые действия. Данные агрегируются в облачное хранилище, анализируются и визуализируются для конечных пользователей.

## Функции изделия
- Сбор и передача облака точек, отражающих положение футболиста в реальном времени
- Построение тепловой карты перемещений каждого игрока
- Формирование облака точек в реальном времени
- Расчёт игровых метрик:
  - общая дистанция, пройденная за матч/тренировку
  - количество ускорений и рывков
  - средняя и максимальная скорость
  - нагрузка на отдельные части тела
  - динамика утомляемости
- Анализ игровых ситуаций (например, участие в атаке, защитные действия, позиционное расположение)
- Визуализация данных в интерфейсе (графики, карты перемещений, таблицы)
- Экспорт данных для тренерского и медицинского штаба

## Характеристики пользователей
- **Тренеры** - используют систему для анализа тактики и нагрузки игроков
- **Тренировочный штаб** - контролируют уровень нагрузки и восстановление футболистов
- **Врачи** - отслеживают показатели, связанные с риском травм и состоянием здоровья
- **Игроки** - могут просматривать персональные отчёты о своей физической активности

## Ограничения
- Требуется стабильное беспроводное соединение для передачи данных в реальном времени
- Система работает только при наличии всех датчиков на игроке
- Обработка больших объёмов данных может требовать высокопроизводительных серверов
- Работа системы ограничена зоной покрытия беспроводной связи (например, стадионом)
- Влияние погодных условий (дождь, снег) на точность датчиков
- Ограничение времени автономной работы сенсоров (зависит от ёмкости батареи)
- Возможные задержки в передаче данных при перегрузке сети

## Предположения и зависимости
- Игроки всегда выходят на поле с полным комплектом датчиков
- Матч проводится в условиях, обеспечивающих бесперебойную передачу данных
- Система интегрируется с существующими аналитическими платформами клуба
- Надёжность работы датчиков зависит от условий эксплуатации (погода, удары, контактная игра)

## 3.2. Надежность

**Доступность:** 99,5% в течение игрового дня (матч и тренировка).

**Среднее время наработки на отказ (MTBF):** Не менее 720 часов непрерывной работы в режиме сбора данных.

**Среднее время восстановления (MTTR):** Не более 30 минут в случае сбоя программного обеспечения.

**Точность:**

- Погрешность определения местоположения игрока не должна превышать 1 метра.
- Погрешность расчета пройденной дистанции не должна превышать 3%.
- **Точность определения нарушений:** Не менее 85% для настроенных правил.

**Макс. количество ошибок:** Не более 1 критического сбоя, приводящего к полной остановке системы, за спортивный сезон.

## 3.3. Производительность

**Время срабатывания:**

- Задержка между сбором данных с датчика и их отображением в интерфейсе не должна превышать 10 секунд.
- Формирование отчета по итогам матча должно занимать не более 2 минут.
- Задержка определения нарушений в реальном времени: < 5 секунд.
- Время отклика интерфейса при работе с командами: < 3 секунд.

**Пропускная способность:**

- Система должна одновременно обрабатывать данные не менее чем с 50 источников.
- Система должна стабильно работать при передаче данных с частотой 10 Гц от каждого датчика.

**Утилизация ресурсов:**

- Пиковая загрузка CPU серверов обработки данных не должна превышать 80%.
- Использование оперативной памяти должно оставаться в пределах 70% от доступного объема.

## 3.4. Ремонтопригодность

- Система должна быть модульной. Сбой в одном модуле (например, расчете одной метрики) не должен приводить к отказу всей системы.
- Должны быть предусмотрены инструменты для мониторинга состояния системы и логирования всех критических операций и ошибок.
- Процедура развертывания обновлений должна быть автоматизирована и не требовать длительного простоя системы.

## 3.5. Ограничения проекта

- Срок разработки и внедрения системы — не более 12 месяцев.
- Бюджет на разработку программного обеспечения и закупку тестового комплекта аппаратуры ограничен.
- Система должна использовать существующие протоколы беспроводной связи (Wi-Fi, Bluetooth LE) и не требовать лицензирования специфических частот на стадионах.

## 3.6. Требования к пользовательской документации

- Должно быть предоставлено **Руководство пользователя**, включающее пошаговые инструкции по работе с веб-интерфейсом для всех категорий пользователей.
- Должно быть предоставлено **Техническое руководство по установке и настройке** системы для администраторов.
- Должно быть предоставлено **Руководство по эксплуатации датчиков**, включающее инструкции по зарядке, креплению и диагностике.

## 3.7. Используемые компоненты

**Аппаратные компоненты:**

- Носимые датчики движения и физиологической активности (например, модели типа Catapult OptimEye S5 или аналог).
- Станции для приема данных (Wi-Fi/Bluetooth антенны) по периметру стадиона.
- Серверное оборудование для развертывания облачной платформы.

**Программные компоненты:**

- Операционная система сервера: Ubuntu Server 20.04 LTS.
- Система управления базами данных: PostgreSQL.
- Бэкенд-фреймворк: Python Django / Node.js.
- Фронтенд-фреймворк: React.

## 3.8. Интерфейсы

- **Пользовательский интерфейс:** Веб-интерфейс должен гарантировать стабильность отображения и отзывчивость в браузерах Chrome, Firefox, последних версий.
- **Программные интерфейсы (API):** Система должна предоставлять REST API для **постепенного изменения** и интеграции с внешними системами.
- **Интерфейсы коммуникаций:** Поддержка протоколов связи с возможностью **постепенного расширения** для поддержки новых стандартов.

## 3.9. Требования к строительству

Не применимо, так как система является программно-аппаратным комплексом и не требует строительных работ.
